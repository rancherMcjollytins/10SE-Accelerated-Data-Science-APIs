#   Requirements Definition

##  Functional Requirements
### Use APIs to get data from database
### UI for users that abstracts inner workings/back-end code in visual graphics
### Functionality without errors and fallback option in case of issues in code.
### Displays data from database (like [_PLACEHOLDER_]) in ways for user to read.
---

##  Non-Functional Requirements
### Automatically installs dependencies
    
### Aesthetically pleasing, intuitive UX (with tools like customtkinter)

### Efficient transfer of data from database and API to user.

### Different formats for displaying data as to be readable and customisable for user preferences
